# GameLab — Полное описание проекта для пояснительной записки

## 1. Общая информация

**GameLab** — это веб-платформа (маркетплейс) для покупки и распространения игровых ассетов, публикации игр, обучения геймдеву и общения разработчиков в форуме. Проект реализован как клиент-серверное SPA-подобное приложение с полноценным REST API и реляционной базой данных.

**Технологический стек (реализованный):**
- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3 (CSS Variables, Grid, Flexbox)
- **Backend:** Node.js + Express.js 4.21
- **База данных:** SQLite3 (реляционная, файл-based)
- **Аутентификация:** JWT (jsonwebtoken 9.0.2) + bcryptjs для хеширования паролей
- **CORS:** Настроен для кросс-доменных запросов
- **Окружение:** dotenv для конфигурации, nodemon для разработки
- **Порт API:** 4000, статические файлы отдаются из корня проекта

## 2. Архитектура страниц

### 2.1 Главная страница — Каталог ассетов (`index.html`)
- **Назначение:** Основной маркетплейс игровых ассетов с фильтрацией и сортировкой
- **Компоненты:**
  - Единая шапка (hero) с логотипом GameLab, поиском, навигацией (Games, Forum, Study), аккаунтом
  - Боковая панель фильтров по типам ассетов (3D Models, Environments, Characters, VFX, Audio, UI, Tools, Animations)
  - Сетка из 16 карточек ассетов с тегами, изображениями, описаниями, списками фич, ценами и кнопками "View"
  - Сортировка (Featured, Price ↑↓, Rating ↑↓)
  - Подсчёт результатов ("Showing X of 16")
- **Скрипты:** `i18n.js`, `asset-data.js`, `search-filters.js`
- **CSS:** `styles.css` (общие стили)

### 2.2 Страница ассета (`asset.html`)
- **Назначение:** Детальный просмотр отдельного ассета с полным описанием, рейтингом и комментариями
- **Компоненты:**
  - Hero-карточка с изображением, типом, названием, вступлением, полным описанием
  - Мета-сетка: Цена, Рейтинг, Совместимость, Размер файла, Последнее обновление
  - Список особенностей (features)
  - Секция рейтинга с визуальным распределением по звёздам (1-5)
  - Секция комментариев с формой ввода и списком
- **Скрипты:** `i18n.js`, `asset-data.js`, `asset-details.js`
- **CSS:** `styles.css`

### 2.3 Страница игр — Showcase (`games.html`)
- **Назначение:** Витрина опубликованных игр с рейтингами и кнопками "Details"
- **Компоненты:**
  - Интро с кнопкой "Publish your game"
  - Сетка из 6 игровых карточек (Cyber Odyssey, Kingdoms of Ash, Neon Drift, Starbound Survivors, Pixel Dungeon Quest, Shadow Protocol)
  - Каждая карточка: превью, название, рейтинг, описание, размеры архива/установки, кнопка Details
- **Скрипты:** `i18n.js`
- **CSS:** `styles.css`, `games.css`

### 2.4 Страница деталей игры (`game-detail.html`)
- **Назначение:** Полное описание отдельной игры с системными требованиями, оценкой и комментариями
- **Компоненты:**
  - Hero-карточка: изображение, жанр (badge), название, intro, полное описание
  - Мета-сетка: Издатель, Рейтинг, Дата выхода, Размер архива, Размер установки, Платформа, Мультиплеер
  - Список особенностей (features)
  - Кнопка "Download"
  - **Секция системных требований:** 2 колонки (Minimum / Recommended) — OS, CPU, RAM, GPU, Storage
  - **Секция скриншотов:** сетка из 4 изображений
  - **Интерактивный рейтинг:** кликабельные звёзды с hover-эффектом
  - **Сводка рейтингов:** общая оценка + распределение по звёздам (bar-chart)
  - **Комментарии:** форма ввода + список отзывов с оценками
- **Скрипты:** `i18n.js`, `game-details.js`
- **CSS:** `styles.css`, `game-detail.css`

### 2.5 Страница авторизации (`auth.html`)
- **Назначение:** Вход и регистрация пользователей
- **Компоненты:**
  - Табы Login / Register
  - Форма входа: email, пароль (с toggle видимости), "Remember me", "Forgot password"
  - Форма регистрации: имя, email, пароль, согласие с политикой
  - Социальные кнопки: Google, Discord, GitHub (визуальные заглушки)
  - Обратная связь через feedback-элементы
- **Скрипты:** `i18n.js`, `auth.js`
- **CSS:** `auth.css`

### 2.6 Восстановление пароля (`forgot-password.html`)
- **Назначение:** Форма отправки ссылки для сброса пароля
- **Компоненты:** Поле email, кнопка "Send Reset Link", ссылка на вход
- **Скрипты:** `i18n.js`, `forgot-password.js`
- **CSS:** `styles.css`, `auth.css`, `forgot-password.css`

### 2.7 Профиль пользователя (`profile.html`)
- **Назначение:** Управление аккаунтом, биографией, просмотр истории просмотров
- **Компоненты:**
  - Панель аккаунта: имя, email
  - Панель настроек: выбор языка (Русский/English), переключатель темы (Dark/Light)
  - Панель биографии: textarea с счётчиком символов (0/300), кнопка "Save bio"
  - Сетка недавно просмотрных ассетов (мини-карточки с изображениями)
  - Кнопка "Log out"
- **Скрипты:** `i18n.js`, `profile.js`
- **CSS:** `styles.css`, `profile.css`

### 2.8 Форум (`forum.html`)
- **Назначение:** Общий чат для общения разработчиков
- **Компоненты:**
  - Боковая панель: список онлайн-пользователей с индикаторами
  - Основной чат: заголовок "General Chat", статус "Live", лента сообщений (автор, время, текст)
  - Форма ввода сообщений с кнопкой "Send"
- **Скрипты:** `i18n.js`, `forum.js`
- **CSS:** `styles.css`, `forum.css`

### 2.9 Учебные материалы (`study.html`)
- **Назначение:** Каталог ресурсов для изучения геймдева
- **Компоненты:**
  - Навигация-фильтр: All, Engines, Documentation, Video Tutorials, Communities, Tools
  - 22 карточки ресурсов с иконками (эмодзи), описаниями и тегами
  - Категории: Unity, Unreal Engine 5, Godot, документация, YouTube-каналы, сообщества, инструменты (Blender, GIMP, Audacity, Figma)
- **Скрипты:** `i18n.js`, `study.js`
- **CSS:** `styles.css`, `study.css`

### 2.10 Публикация игры (`publish.html`)
- **Назначение:** Форма отправки игры на модерацию
- **Компоненты:**
  - **Basic Information:** название, описание (счётчик символов), жанр (select из 11 вариантов)
  - **Files & Media:** архив (ZIP/RAR/7Z), обложка, скриншоты (до 5 файлов)
  - **System Requirements:** минимальные и рекомендуемые (OS, CPU, RAM, GPU, Storage, Additional Notes)
  - **Additional Information:** движок, ссылка на проект, ссылка на трейлер, согласие с правилами
  - Кнопка "Submit for Moderation" с валидацией и feedback
- **Скрипты:** `i18n.js`, `publish.js`
- **CSS:** `styles.css`, `publish.css`

## 3. Серверная часть (Backend API)

### 3.1 Структура сервера
```
server/
├── server.js          — Точка входа, Express-приложение
├── db.js              — Обёртка над SQLite (run, get, all, initDb)
├── .env               — Переменные окружения (PORT, JWT_SECRET, CLIENT_ORIGIN)
├── database.sqlite    — Файл базы данных
├── middleware/
│   └── auth.js        — JWT-мидлвар для защиты маршрутов
├── routes/
│   ├── auth.js        — Регистрация, вход, получение пользователя
│   ├── profile.js     — Профиль, обновление био, история просмотров
│   └── assets.js      — Трекинг просмотров ассетов
└── package.json       — Зависимости
```

### 3.2 База данных (SQLite)
**Таблицы:**
- `users` — id, email (unique), username, password_hash, created_at
- `profiles` — user_id (FK → users), bio, avatar_url, updated_at
- `asset_views` — id, user_id (FK), asset_id, viewed_at (с индексом)

### 3.3 API Endpoints

| Метод | Путь | Описание | Auth |
|-------|------|----------|------|
| `GET` | `/api/health` | Проверка работоспособности | Нет |
| `POST` | `/api/auth/register` | Регистрация нового пользователя | Нет |
| `POST` | `/api/auth/login` | Аутентификация, возврат JWT | Нет |
| `GET` | `/api/auth/me` | Получение данных текущего пользователя | JWT |
| `GET` | `/api/profile/me` | Профиль с bio и avatar | JWT |
| `PATCH` | `/api/profile/me` | Обновление bio и avatar_url | JWT |
| `GET` | `/api/profile/me/recent-views` | История просмотров ассетов | JWT |
| `POST` | `/api/assets/:assetId/view` | Трекинг просмотра ассета | JWT |

### 3.4 Аутентификация
- JWT-токен с expiresIn: "7d", subject = user_id
- Мидлвар `authRequired` извлекает токен из заголовка `Authorization: Bearer <token>`
- Токен сохраняется в `localStorage` как `gamelab_token`
- Данные пользователя: `gamelab_user`

## 4. Система интернационализации (i18n.js)

- **Языки:** Русский (ru) и Английский (en)
- **Хранение:** `localStorage` ключ `gamelab_lang`
- **Функции:**
  - `getLang()` — получение текущего языка
  - `setLang(lang)` — установка языка
  - `t(key)` — получение перевода по ключу
  - `applyTranslations()` — массовый перевод всех элементов на странице по селекторам
- **Переводимые элементы:** навигация, поиск, заголовки, описания ассетов, описания игр, жанры, фичи, системные требования, формы, футер, форум, учебные материалы, страница публикации
- **Словарь:** ~250+ ключей перевода для каждой страницы

---

## 5. Система тем

- **Темы:** Тёмная (dark, по умолчанию) и Светлая (light)
- **Хранение:** `localStorage` ключ `gamelab_theme`
- **Реализация:** CSS Variables (`:root` для dark, `html[data-theme="light"]` для light)
- **Функции:** `getTheme()`, `setTheme()`
- **Применение:** На каждой странице через `data-theme` атрибут на `<html>`

```
  └── profile.html (профиль пользователя)
```

## 8. Ключевые характеристики проекта

- **10 уникальных страниц** с собственными стилями и логикой
- **~1250+ строк CSS** в основном файле `styles.css` + 8 дополнительных CSS-файлов
- **~1130+ строк i18n.js** с переводами для всех элементов
- **REST API** с JWT-аутентификацией и SQLite
- **Полная двуязычность** (RU/EN) на всех страницах
- **Адаптивный дизайн** (responsive media queries)
- **Тёмная и светлая темы** с CSS-переменными
- **Модульная архитектура** сервера (middleware, routes, db abstraction)
- **Aria-атрибуты** и семантический HTML для доступности
